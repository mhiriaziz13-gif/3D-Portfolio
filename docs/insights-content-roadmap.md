# Insights content roadmap

No public Insights section should launch until an article is evidence-complete and human-reviewed. A future server-rendered `/insights/[slug]` model should contain author, published date, meaningful update date, summary, body, references, related projects, canonical URL and social image. Drafts stay outside public routes.

Evidence: barbershop project; confirm approved artifacts; link project/expertise; cite Google Business Profile guidance; CTA: contact; type: Article; annual review.

Evidence: e-learning contribution; confirm architecture and responsibilities; link project; cite Ollama and framework documentation; CTA: view GitHub; type: TechArticle; review on dependency changes.
